O que é o AngularJS

Um framework para criar aplicações web dinâmicas, de maneira rápida, testável e de fácil manutenção.

Princípios do AngularJS

  • Desenvolvimento Rápido
  • Escrever menos código
  • Modularidade
  • Pensado para ser testado

O que o AngularJS oferece

  • Controllers
  • Templates
  • Two-Way data bindings
  • Services
  • Directives
  • Integração com REST
  • Injeção de dependencias
  • Transições (em CSS3)

Um exemplo bem simples


      <!doctype html>
      <html ng-app>
        <head>
          <script src="../js/angular.min.js"></script>
        </head>
        <body>
          <form>
            <label>Seu nome?</label>
            <input type="text" ng-model="name" placeholder="Seu nome?">
          </form>
          <h3>Meu nome é {{ name }}!</h3>
        </body>
      </html>

Controllers

São funções usadas para "aumentar" a instância do $scope. Adicionando um valor ou um comportamento a um objeto.


    function TodoCtrl($scope) {
      $scope.todos = [
        { text: 'Aprender AngularJS', done: true },
        { text: 'Criar um novo App', done: false }
      ];

      $scope.addTodo = function() {
        if ($scope.todoText) {
          $scope.todos.push({ text: $scope.todoText, done: false });
          $scope.todoText = '';
        }
      };
    };

Views

As views no angular são simplesmente HTML5.
É onde toda a presentation logic deve estar.


      <div ng-controller="TodoCtrl">
        <ul>
          <li ng-repeat="todo in todos">
            <input type="checkbox" ng-model="todo.done">
            <span class="done-{{ todo.done }}">{{ todo.text }}</span>
          </li>
        </ul>
        <form ng-submit="addTodo()">
          <input type="text" ng-model="todoText">
          <button type="submit">Add</button>
        </form>
      </div>

Resultado

Two-Way data binding

Data binding no angular funciona de maneira bi-directional, qualquer alteração na View se reflete no modelo e vice-versa.
